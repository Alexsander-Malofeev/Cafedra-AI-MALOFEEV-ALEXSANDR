## 2. Protocol

**Разбиение:** train/test (75%/25%, random_state=42, stratify=y)

**Подбор:** CV на train (5 фолдов, StratifiedKFold, оптимизация ROC-AUC через GridSearchCV)

**Метрики:** accuracy, F1, ROC-AUC
- **Accuracy:** базовая метрика, но вводит в заблуждение при дисбалансе
- **F1-score:** важна для дисбаланса, баланс precision/recall
- **ROC-AUC:** основная метрика для сравнения, устойчива к дисбалансу

## 6. Conclusion

3-6 коротких тезисов: что вы поняли про деревья/ансамбли и про честный ML-протокол.

1. **Контроль сложности деревьев:** DecisionTree без ограничений легко переобучается. Параметры max_depth=7 и min_samples_leaf=10 помогли найти баланс между bias и variance, но дерево все равно уступает ансамблям.

2. **Эффективность ансамблей:** RandomForest (bagging) показал лучший ROC-AUC (0.8989), демонстрируя преимущество уменьшения variance через агрегацию множества деревьев. GradientBoosting показал лучший F1-score (0.6890), подтверждая эффективность последовательного улучшения.

3. **Метрики при дисбалансе:** Accuracy (96.7%) вводит в заблуждение при дисбалансе 95%/5%. ROC-AUC и F1-score необходимы для адекватной оценки. RandomForest выбран победителем по ROC-AUC, а не по accuracy.

4. **Честный ML-протокол:** Фиксированный random_state=42 обеспечил воспроизводимость. Стратификация сохранила распределение классов. CV на train с GridSearchCV предотвратил data leakage. Test использован один раз для финальной оценки.

5. **Устойчивость результатов:** 5 прогонов RandomForest с разными random_state показали стабильность (CV=0.93%), что подтверждает надежность вывода о его эффективности.

6. **Интерпретируемость vs производительность:** Ансамбли менее интерпретируемы, чем одиночные деревья, но permutation importance позволила выявить ключевые признаки. Компромисс между интерпретируемостью и точностью оправдан для данной задачи.

**Общий вывод:** Для задач с сильным дисбалансом классов ансамблевые методы (особенно RandomForest) предпочтительнее одиночных моделей. Честный ML-протокол с фиксированным random_state, стратификацией, CV на train и однократным использованием test обеспечивает достоверные и воспроизводимые результаты.
