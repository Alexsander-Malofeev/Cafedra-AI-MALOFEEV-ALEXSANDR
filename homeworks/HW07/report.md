## 2. Protocol

### 2.1 Препроцессинг
Для всех датасетов применен единый пайплайн:
1. **Imputation:** SimpleImputer со стратегией 'median' (для обработки возможных пропусков)

3. Все датасеты содержали только числовые признаки, категориальные признаки отсутствуют

### 2.2 Поиск гиперпараметров
**KMeans:**
- Диапазон k: от 2 до 20 кластеров
- Критерий выбора: максимизация silhouette score
- Фиксированные параметры: random_state=42, n_init=10

**DBSCAN:**
- Диапазон eps: от 0.1 до 3.0 с шагом 0.2
- min_samples: фиксировано на 5
- Критерий выбора: максимизация silhouette score (среди вариантов с ≥2 кластерами)
- Учет шума: метрики считались только для не-шумовых точек

**AgglomerativeClustering:**
- Диапазон k: от 2 до 10
- Методы linkage: ['ward', 'complete', 'average']
- Критерий выбора: максимизация silhouette score

### 2.3 Метрики качества
Для всех методов вычислялись:
1. **Silhouette Score** (больше = лучше) - оценка компактности и разделимости кластеров
2. **Davies-Bouldin Score** (меньше = лучше) - оценка среднего сходства кластеров
3. **Calinski-Harabasz Score** (больше = лучше) - отношение дисперсии между кластерами к дисперсии внутри

**Для DBSCAN:**
- Метрики вычислялись только для точек, не помеченных как шум (-1)
- Отдельно фиксировалась доля шумовых точек

### 2.4 Визуализация

- PCA(2D) для визуализации результатов кластеризации
- Объясненная дисперсия PCA: от 30% до 80% в зависимости от датасета

## 6. Conclusion

1. **Масштабирование обязательно** - без StandardScaler distance-based методы дают некорректные результаты из-за разных шкал признаков

2. **Выбор метода зависит от структуры данных** - нет универсального лучшего алгоритма:
   - KMeans хорош для сферических кластеров
   - DBSCAN для данных с выбросами/шумом
   - Agglomerative для иерархических структур

3. **Silhouette score** - эффективная метрика для подбора числа кластеров и сравнения методов, но требует осторожной интерпретации при наличии шума

4. **PCA визуализация** - необходимый минимум для оценки качества кластеризации, но может скрывать многомерные структуры

5. **Устойчивость важна** - проверка на разных random_state показала, что для некоторых данных KMeans дает стабильные результаты

6. **Множество метрик лучше одной** - использование silhouette, Davies-Bouldin и Calinski-Harabasz дает более полную картину

7. **Обработка шума в DBSCAN** - требует отдельного рассмотрения при вычислении метрик

8. **Экспериментальный протокол** - последовательность EDA → препроцессинг → подбор параметров → оценка → визуализация обеспечивает воспроизводимость и достоверность результатов
