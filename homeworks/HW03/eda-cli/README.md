# eda-cli: Мини-приложение для EDA на Python

Это проект для домашнего задания HW03. Он предоставляет CLI-инструмент для базового анализа данных (EDA) с использованием Pandas и Matplotlib.

## Установка
Перейдите в папку проекта и выполните:
uv sync

## Доступные команды CLI
- `eda-cli overview <файл.csv>`: Выводит обзор датасета, включая размеры, качество данных и список проблем (пропуски, дубликаты, константные колонки, высокая кардинальность).
- `eda-cli report <файл.csv> [опции]`: Генерирует отчёт в формате Markdown (report.md) с основными характеристиками, проблемами и гистограммами для числовых колонок. Отчёт сохраняется в указанной папке (--out-dir).

## Новые параметры для команды report (добавлены в HW03)
- `--max-hist-columns <N>`: Ограничивает количество числовых колонок, для которых строятся и сохраняются гистограммы в PNG-файлах (по умолчанию: 4). Влияет на визуализации в отчёте — меньше колонок = меньше файлов.
- `--min-missing-share <X>`: Порог доли пропусков (от 0 до 1), выше которого колонка считается проблемной и добавляется в отдельный список в разделе "Обнаруженные проблемы" отчёта (по умолчанию: 0.3). Также влияет на расчёт quality_score (дополнительный штраф).

Эти параметры передаются в функции анализа и визуализации, и их значения упоминаются в отчёте.

## Пример вызова команды report с новыми опциями
uv run eda-cli report data/example.csv --out-dir reports_example --max-hist-columns 2 --min-missing-share 0.1

Это создаст отчёт с гистограммами только для первых 2 числовых колонок и отметит проблемными те колонки, где пропусков >10%.
